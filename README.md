# Rust Utility Overlay

Десктопное Python-приложение — полупрозрачный оверлей поверх игры **Rust** с набором полезных инструментов. Работает в фоне, вызывается глобальным хоткеем, не требует переключения из игры.

**Целевая платформа:** Windows  
**Игра:** Rust (vanilla, soft-side рейд)

---

## Быстрый старт

```powershell
pip install -r requirements.txt
python main.py
```

| Клавиша | Действие |
|---------|----------|
| **F5** | Показать / скрыть оверлей |
| **F6** | Полный выход из приложения |

Данные сессии (рейд, крафт, заметки, таймеры) хранятся в `%APPDATA%\RustUtilityOverlay\session.json`.

> **Важно:** библиотека `keyboard` перехватывает F5/F6 глобально. Если хоткеи не срабатывают в полноэкранной игре — запустите приложение **от имени администратора**.

Рекомендуемый режим Rust: **оконный** или **безрамочный** — оверлей (`topmost`) корректно отображается поверх игры.


Готовый файл: **`dist\RustUtilityOverlay.exe`** (~29 МБ). Отправьте друзьям только его.

**Что нужно друзьям:**

- Windows 10/11 (64-bit)
- Python **не нужен**

Данные каждого пользователя сохраняются отдельно в `%APPDATA%\RustUtilityOverlay\`.

## Текущие функции

Оверлей содержит **8 вкладок**. Данные рейда, крафта, заметок и таймеров сохраняются в `data/session.json` между запусками.

### 1. Калькулятор рейда (`raid_calculator`)

Считает количество взрывчатки и стоимость крафта для рейда базы. Референсы: [rusttips.com](https://rusttips.com/rust-raid-calculator/), [rustexplore.com](https://rustexplore.com/ru/tools/raid-calculator).

**Что умеет:**

- Добавлять несколько строений в список рейда
- Автоматически подбирать самый дешёвый вариант **по сере** (чекбокс «Авто»)
- Вручную выбирать тип взрывчатки
- Показывать таблицу: строение → кол-во → взрывчатка → кол-во → время → ресурсы
- Суммировать итоговые ресурсы и общее время рейда
- Удалять отдельные строки или очищать весь список
- **Сохранять список рейда** между запусками приложения

**Поддерживаемые строения (26):**

| Категория | Объекты |
|-----------|---------|
| Стены | Деревянная, каменная, металлическая, бронированная |
| Двери | Деревянная, листовой металл, гаражная, бронированная, люк, витрина |
| Внешние стены | Высокая деревянная, высокая каменная |
| Ловушки / турели | Автотурель, дробовая ловушка, огнемёт, Зенитка |
| Объекты | Шкаф, баррикада, забор, решётка, окна, верстаки 1–3, торговый автомат |

**Поддерживаемая взрывчатка (6):**

| ID | Название |
|----|----------|
| `c4` | Взрывчатка с таймером |
| `rocket` | Ракета |
| `satchel` | Связка бобовых гранат (сатчел) |
| `explosive_ammo` | Патрон 5.56 (разрывной) |
| `beancan` | Бобовая граната |
| `hv_rocket` | Ракета высокоскоростная (только дерево / мягкие цели) |

**Пример:** каменная стена — 2 C4 (4400 серы) или 4 ракеты (5600 серы).

### 2. Калькулятор крафта (`craft_calculator`)

Считает ресурсы для крафта выбранной взрывчатки. Показывает итог в **переработанном** и **сыром** виде.

**Что умеет:**

- Выбрать тип взрывчатки и количество
- Добавить несколько позиций в список (например, 4 C4 + 10 ракет)
- Показать **переработанные ресурсы**: сера, фрагменты металла, уголь, НКТ
- Показать **сырьё**: серная руда, металлическая руда, дерево, сырая нефть
- Показать **прочие компоненты**: HQM, ткань, микросхемы, трубы и т.д.
- Показать блок **«Плавка в печи»**: дерево на руду, уголь с плавки, доп. дерево, время
- **Сохранять список крафта** между запусками приложения

**Коэффициенты переработки (vanilla):**

| Сырьё | → | Переработанное | Коэффициент |
|-------|---|----------------|-------------|
| Серная руда | → | Сера | 1:1 |
| Металлическая руда | → | Фрагменты металла | 1:1 |
| Дерево | → | Уголь | ~75% при сжигании в печи |

Данные о стоимости крафта взрывчатки переиспользуются из `features/raid_calculator/data.py` (`CraftCost`). Плавка — из `features/furnace_calculator/`.
| Сырая нефть | → | НКТ | 2:1 |

### 3. Калькулятор печи (`furnace_calculator`)

Отдельная вкладка для расчёта плавки руды в стандартной печи. Логика общая с блоком «Плавка» в калькуляторе крафта.

**Что умеет:**

- Ввести количество металлической, серной и HQM руды
- Указать дополнительную потребность в угле
- Показать дерево на плавку, уголь с плавки (~75%), доп. дерево при нехватке угля
- Показать время плавки и результат (сера, металл, HQM, уголь)
- Указать **количество печей** — время делится на число печей, расход дерева не меняется

**Коэффициенты печи (vanilla):**

| Руда | Дерево на 1 руду | Время плавки |
|------|------------------|--------------|
| Металлическая | 5 | 5 сек |
| Серная | 2.5 | 2.5 сек |
| HQM | 10 | 15 сек |

Уголь с плавки: **~75%** от сожжённого дерева. Если угля с плавки не хватает для крафта — считается дополнительное дерево.

### 4. Экскаватор / карьер / качалка (`resource_machines`)

Калькулятор дизельных машин: дизель → время работы → выход ресурсов.

**Что умеет:**

- Выбрать машину (экскаватор по сере/камню/HQM, карьер, качалка нефти)
- Указать количество дизеля и добавить несколько запусков
- Показать время и суммарный выход (сера, камень, HQM, нефть)

### 5. Заметки / коды дверей (`notes`)

Локальный блокнот для баз и рейдов.

**Что умеет:**

- Записывать название, код двери и заметку
- Хранить список записей между запусками
- Удалять отдельные записи или очищать всё

### 6. Таймеры (`timers`)

Фоновые таймеры с уведомлением (звук + показ оверлея).

**Типы таймеров:**

- **Напоминание** — произвольный текст и время в минутах (например, «проверить печи»)
- **Декей** — время разрушения построек без upkeep (дерево, камень, металл, броня)
- **Машины** — экскаватор / карьер / качалка по количеству дизеля

Активные таймеры сохраняются в сессии и восстанавливаются при перезапуске.

### 7. Генетика растений (`genetics`)

Калькулятор кроссбридинга: центральное растение + 4 соседа.

**Что умеет:**

- Ввести гены (G/Y/H/W/X) для центра и соседей
- Рассчитать результат скрещивания по весам генов
- Показать пояснение по каждому слоту

### 8. Калькулятор электрики (`electricity`)

Баланс мощности (rW) для солнечных панелей, ветряков, турелей, батарей и т.д.

**Что умеет:**

- Указать количество источников, потребителей и батарей
- Показать генерацию, потребление, баланс и суммарную ёмкость батарей

## Архитектура

```
main.py
  │
  ├── keyboard (глобальные хоткеи F5/F6)
  ├── SessionStore (storage/session.py) → data/session.json
  ├── TimerManager (services/timer_manager.py)
  │
  └── OverlayWindow (overlay/window.py)
        ├── боковое меню функций
        └── Feature[] — вкладки с инструментами
              ├── RaidCalculatorFeature
              ├── CraftCalculatorFeature
              ├── FurnaceCalculatorFeature
              ├── ResourceMachinesFeature
              ├── NotesFeature
              ├── TimersFeature
              ├── GeneticsFeature
              └── ElectricityFeature
```

### Поток запуска

1. `main.py` создаёт `SessionStore`, `OverlayWindow`, `TimerManager` и список `Feature`-объектов
2. `OverlayWindow` строит UI: top bar + sidebar + content area
3. Для каждой функции вызывается `feature.build(frame)` — UI создаётся один раз при старте
4. Глобальный хоткей F5 вызывает `overlay.toggle()` через `root.after(0, ...)` — потокобезопасно для Tkinter
5. При старте окно **скрыто** (`withdraw`), пользователь видит только консольное сообщение

### Система функций (`Feature`)

Все инструменты оверлея — классы, наследующие `features/base.py → Feature`:

```python
class Feature(ABC):
    id: str       # уникальный идентификатор (напр. "raid_calculator")
    title: str    # название в боковом меню

    def build(self, parent: ctk.CTkFrame) -> None  # создать UI (обязательно)
    def on_show(self) -> None   # при переключении на вкладку (опционально)
    def on_hide(self) -> None   # при уходе с вкладки (опционально)
```

**Добавление новой функции — 3 шага:**

1. Создать пакет `features/my_feature/` с классом `MyFeature(Feature)`
2. Реализовать `build()` — разместить виджеты в `parent`
3. Добавить экземпляр в список `features` в `main.py`:

```python
features = [
    RaidCalculatorFeature(session),
    CraftCalculatorFeature(session),
    FurnaceCalculatorFeature(),
    MyFeature(),  # ← новая функция
]
overlay.set_features(features)
```

Новая вкладка появится в sidebar автоматически. Логику держите отдельно от UI (как в `raid_calculator/`).

## Калькулятор рейда — как работает изнутри

### Модель данных (`features/raid_calculator/data.py`)

Три ключевых dataclass:

**`Structure`** — строение для рейда:
```python
Structure(id="stone_wall", name="Стена из камня", hp=500, category="walls")
```

**`Explosive`** — тип взрывчатки:
```python
Explosive(
    id="c4",
    name="Взрывчатка с таймером",
    craft=CraftCost(sulfur=2200, metal_fragments=200, ...),
    damage={"stone_wall": 275, "metal_wall": 275, ...},  # урон по structure_id
    raid_seconds=10,
)
```

**`CraftCost`** — стоимость крафта одной единицы взрывчатки. Поддерживает `+` (сложение) и `.scale(count)` (умножение).

### Формула расчёта (`features/raid_calculator/calculator.py`)

```
кол-во_взрывчатки = ceil(HP_строения / урон_взрывчатки_по_этому_строению)
общее_кол-во = кол-во_взрывчатки × кол-во_строений
стоимость = craft_рецепт.scale(общее_кол-во)
время = raid_seconds × общее_кол-во
```

**Авто-режим:** функция `best_explosive_for()` перебирает все `EXPLOSIVES`, для каждого считает итоговую серу и выбирает минимум.

**Ручной режим:** пользователь выбирает `explosive_id`, расчёт идёт по той же формуле.

**Агрегация:** `calculate_raid(entries)` суммирует `CraftCost` и время по всем строкам таблицы.

### UI (`features/raid_calculator/widget.py`)

`RaidCalculatorFeature` хранит список `RaidEntry` в `self._entries`. При каждом добавлении/удалении вызывается `_refresh_table()`, которая:
- перерисовывает строки таблицы
- вызывает `calculate_raid()` для блока «Итого»

## Оверлей — технические детали

`OverlayWindow` (`overlay/window.py`):

- `overrideredirect(True)` — без системной рамки (нет кнопок закрыть / свернуть / развернуть)
- `resizable(False, False)` — пользователь не растягивает окно вручную
- `fit_to_content()` — автоматический подбор размера под контент активной вкладки; вызывается при смене вкладки и после `feature.request_resize()`
- `attributes("-topmost", True)` — окно поверх всех, включая игру
- `attributes("-alpha", 0.94)` — лёгкая прозрачность
- `withdraw()` / `deiconify()` — скрытие без завершения процесса
- `ctypes.windll.user32` — флаг `WS_EX_LAYERED` для корректного layered-окна на Windows
- Потокобезопасность: хоткеи из `keyboard` вызывают UI только через `root.after(0, callback)`

## Ограничения и известные нюансы

- **Только Windows** — хоткеи и layered-окно завязаны на WinAPI
- **Soft-side рейд** — урон считается по мягкой стороне строений (стандарт для калькуляторов)
- **Данные статические** — HP и урон захардкожены в `data.py`, не подтягиваются из игровых файлов автоматически. При патче Rust нужно обновить `data.py` вручную
- **Авто-режим оптимизирует только серу** — не учитывает доступность верстака, время, HQM и т.д.
- **HV-ракеты** — урон = 0 для камня/металла (как в игре), доступны только для дерева и мягких целей
- **Нет скроллбаров** — таблицы растут вместе с окном; при добавлении строк вызывается `request_resize()`
- **Админ-права** — могут потребоваться для глобальных хоткеев в exclusive fullscreen

## Для AI-агентов: правила работы с репозиторием

### Контекст проекта

Это **утилита-оверлей для Rust**, не чит и не бот. Функции — вспомогательные инструменты (калькуляторы, справочники и т.д.), вызываемые по F5 поверх игры.

### Куда добавлять код

| Задача | Куда |
|--------|------|
| Новый инструмент / вкладка | `features/<name>/` + регистрация в `main.py` |
| Игровые данные (HP, урон, рецепты) | `features/raid_calculator/data.py` |
| Логика расчётов рейда | `features/raid_calculator/calculator.py` |
| Логика крафта / конвертация сырья | `features/craft_calculator/` |
| Логика плавки в печи | `features/furnace_calculator/` |
| UI конкретной функции | `features/<name>/widget.py` |
| Общее окно, навигация, хоткеи | `overlay/window.py`, `main.py` |

### Соглашения

- UI на **CustomTkinter**, тёмная тема, акцентный цвет `#c45c26`
- Логика отделена от UI (dataclass + чистые функции в `calculator.py`, виджеты в `widget.py`)
- `Feature.id` — snake_case, уникальный в пределах проекта
- Не добавлять зависимости без необходимости
- Не ломать потокобезопасность: UI-операции только из main thread Tkinter (`root.after`)
- Тексты UI — на русском

### Что можно добавить в будущем

- Справочник крафта и переработки
- Настройки хоткеев и позиции окна
- ECO-рейд (молотовы, бобовые гранаты без верстака 3)
- Таймер вайпа сервера

### Обновление данных рейда

При изменении баланса Rust:

1. Сверить HP строений и урон взрывчатки с [rustlabs.com](https://rustlabs.com) или [rusttips.com](https://rusttips.com/rust-raid-calculator/)
2. Обновить `STRUCTURES` (HP) и `EXPLOSIVES` (damage, craft) в `data.py`
3. Проверить: `python -c "from features.raid_calculator.calculator import calculate_entry; e = calculate_entry('stone_wall', 1); print(e.explosive_count, e.cost.sulfur)"` → ожидается `2 4400`
